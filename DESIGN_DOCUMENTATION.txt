# Densat Facility Management App - UI/UX Design Documentation

## Theme & Design System

### Color Palette
Based on the app icon (appicon.png), the design uses a professional green color scheme:

**Primary Colors:**
- Primary Green: #398265
- Secondary Green: #14D387
- Gradient: Linear gradient from Primary to Secondary

**Accent Colors:**
- Red (for alerts/hot deals): #EF4E4E
- Blue (for info): #37ABFF

**Neutral Colors:**
- Background: #FFFFFF (White)
- Surface: #F5F6F7 (Light Gray)
- Borders: #DCDFE3
- Text Primary: #242D35
- Text Secondary: #6B7580
- Text Hint: #9BA1A8

### Typography
- Font Family: Google Fonts - Inter
- Heading 1: 28px, Bold (700)
- Heading 2: 20px, Bold (700)
- Body: 15-16px, Medium (500)
- Caption: 13px, Regular (400)

### Design Style
**iOS-Inspired Elements:**
1. Rounded corners (16px default, 20px for larger cards)
2. Subtle shadows (elevation-like effects)
3. Smooth transitions and animations
4. SF Symbols-style icons (using Cupertino icons)
5. Clean, minimal interface with ample white space
6. Large, readable text
7. Card-based layouts
8. Bottom sheet interactions

## User Flow

### Booking Flow:
1. User opens app ‚Üí Splash ‚Üí Home Screen
2. Browse hot deals carousel or categories
3. Tap on service ‚Üí Service Details
4. Tap "Book Now" ‚Üí Date & Time Selection
5. Enter/Select Address ‚Üí Add Notes
6. Review Summary ‚Üí Confirm Booking
7. Booking created with "Pending" status
8. Navigate to Bookings tab to see pending request

### Admin Approval Flow (Simulated):
1. Admin approves booking (background - not in app yet)
2. Booking status changes to "Approved"
3. User sees "Approved" in bookings tab
4. User taps booking ‚Üí "Select Service Provider" button appears

### Service Provider Selection:
1. User taps "Select Service Provider"
2. Sees list of available providers
3. Selects a provider
4. Can chat with provider
5. Can call provider
6. Service progresses to "In Progress"

### Completion:
1. Service completed
2. Status changes to "Completed"
3. User can rate and review
4. Service history maintained

## Design Principles

1. **Consistency**: Uniform spacing, colors, and components throughout
2. **Clarity**: Clear hierarchy and readable text
3. **Feedback**: Visual feedback for all interactions
4. **Efficiency**: Minimal steps to complete tasks
5. **iOS Feel**: Native iOS design patterns and interactions
6. **Professional**: Clean, modern, business-appropriate design
7. **Trust**: Clear pricing, status updates, provider information
